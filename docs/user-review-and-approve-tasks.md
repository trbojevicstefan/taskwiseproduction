# Review And Approve Tasks

Review Tasks is the center of the meeting-to-task workflow.

## Review Queue

Meetings are grouped by:

- Needs review
- Processing
- Failed
- Reviewed

Open a meeting to see its summary, suggested tasks, people, due dates, confidence-related review labels, and supporting details.

## Edit Before Moving Work Forward

Before treating AI output as final, review:

- Title
- Owner
- Due date
- Priority
- Notes or description
- Status

Use task detail when you need source evidence, subtasks, comments, or board movement controls.

## Approve Tasks For The Board

1. Select one or more reviewed tasks.
2. Choose Approve to Board from the selection toolbar.
3. Taskwise marks the selected tasks as approved and adds board cards for them.

Meetings move from Needs review to Reviewed once their tasks no longer need review.

## Bulk Actions

Use selected tasks to assign owners, set due dates, copy/export, share, or push to connected destinations. Failed or processing meetings should be retried or inspected before relying on their task list.
